# Auto-scroll fix — second attempt (more thorough)

The previous fix (position check on every effect run) didn't work. The auto-scroll is still snapping the user back to the bottom during streaming.

## Likely cause

The streaming response re-renders the entire message list on every token. That re-render can reset scroll position regardless of position checks. Just checking "am I near the bottom" on every effect isn't enough — you need to track the user's scroll *intent* across the whole streaming session.

## The fix

Use a state flag to track whether auto-scroll should be active. Disable it when the user scrolls up. Re-enable it when they scroll back to the bottom.

In `artifacts/aiforrhhs/src/pages/Chat.tsx`:

1. Add state for auto-scroll behavior:

```javascript
const [autoScrollEnabled, setAutoScrollEnabled] = useState(true);
const scrollContainerRef = useRef<HTMLDivElement>(null);
```

2. Add a scroll event listener on the scroll container. When the user scrolls, check whether they're near the bottom:

```javascript
useEffect(() => {
  const el = scrollContainerRef.current;
  if (!el) return;
  
  const handleScroll = () => {
    const isNearBottom = el.scrollHeight - el.scrollTop - el.clientHeight < 100;
    setAutoScrollEnabled(isNearBottom);
  };
  
  el.addEventListener('scroll', handleScroll);
  return () => el.removeEventListener('scroll', handleScroll);
}, []);
```

3. Replace the existing message-update auto-scroll effect with one that checks the flag:

```javascript
useEffect(() => {
  if (!autoScrollEnabled) return;
  const el = scrollContainerRef.current;
  if (el) {
    el.scrollTop = el.scrollHeight;
  }
}, [messages, autoScrollEnabled]);
```

4. Make sure `scrollContainerRef` is attached to the actual scrolling container (the chat area, not the outer wrapper). It needs to be on the element that has `overflow-y: auto` or `overflow-y: scroll`.
